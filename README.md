# 오늘의 한잔 — 음주기안 (v1)

오늘 조건에 맞는 술집을 **결재형 UI**로 추천하는 로컬-퍼스트 웹앱.
회원가입·서버 DB 없이, 개인 기록은 전부 브라우저(IndexedDB)에만 저장한다.
공용 업장 풀(서울 316곳)만 객관 정보로 읽기 전용 제공한다.

> 설계 근거: [`오늘의한잔_2단계_설계서_v2.md`](./오늘의한잔_2단계_설계서_v2.md)
> (1단계 프로토타입 원본은 주관 평가 데이터가 박혀 있어 공개 저장소에서 제외함)

---

## 실행

정적 파일이지만 공용 풀을 `fetch`로 읽으므로 **HTTP 서버로 띄워야 한다**
(`file://`로 직접 열면 브라우저가 JSON 로드를 차단함 — 앱이 안내 문구를 표시함).

```bash
cd oneul-hanjan
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000
```

빌드 단계·번들러·의존성 없음. 순수 ES 모듈 + 정적 JSON.

## 데이터 모델

### 공용 풀 — `data/venues.json` (읽기 전용)

객관 정보만. 1단계의 주관 평가(만족도·재방문·좋았던점 등)는 공용 풀에 넣지 않는다.

```json
{
  "meta": { "name": "...", "version": 2, "count": 316, "region": "서울특별시" },
  "venues": [
    { "id": "v001", "name": "강남초장", "cat": "생선회", "gu": "강남구", "dong": "개포동" }
  ]
}
```

### 개인 데이터 — IndexedDB `oneul-hanjan` (기기 로컬)

- **`meta`** 스토어 (key-value): 취향 퀴즈 프로필 등
  ```js
  profile = { sits:[], priorities:[], avoid:[], vibe:[], regions:[], v:1 }
  ```
- **`records`** 스토어 (keyPath `id`, 인덱스 `status`/`gu`): 가본 곳/위시 기록
  ```js
  {
    id, venueId,            // venueId: 공용 풀 업장 참조(있으면)
    name, cat, gu, dong,
    status,                 // '방문함' | '위시'
    revisit, sat, sit,      // 방문함 전용: 재방문('예'|'아니오'), 만족도(0~5), 동행 유형
    good:[], bad:[],        // 우대조건/반려사유 태그 (vocab 토큰)
    note,                   // 메모 (타인 개인정보 입력 금지 안내)
    createdAt, updatedAt
  }
  ```

`good`/`bad` 토큰과 퀴즈·기안 화면의 선택지는 모두 [`vocab.js`](./js/vocab.js)의 동일한 토큰을 쓴다(같은 자[尺] 원칙). **이 토큰이 추천 매칭 키이므로 임의 변경 금지.**

## 추천 로직 — `recommend.js`

후보 = 본인 기록(위시 + 방문함 중 재방문≠아니오) ∪ 공용 풀. 선택한 구 기준으로 합친다.

**출처별 가중치**

| 출처 | 점수 구성 |
| --- | --- |
| 본인 방문함(검증) | 검증 +40, 만족도 ×4, 동행 일치 +12, 우대조건 매칭 ×10, 재방문 예 +10 |
| 본인 위시 | 등재 의사 +22, 우대조건 매칭 ×8, 동행 +6, 카테고리 부합 |
| 공용 풀 | 객관 부합도(성향↔카테고리·우대업종·선호지역) **× 감쇠계수** |

**감쇠계수** = `max(0.3, 1 − 본인등재수 / DECAY_THRESHOLD)`
본인 등재가 늘수록 공용 풀 비중이 줄어, 추천이 점점 개인 데이터 중심으로 이동한다.

**반려(제외)**: 오늘 반려사유와 기록 태그가 겹치거나, 재방문='아니오'인 건은 후보에서 빼고 반려 사유로 표기한다.

**추천 사유**는 공문서체 타인추천형(시스템이 객관 사유를 댄다) — 설계서 6장.

## 개인정보 설계 원칙 (v1)

- 회원가입·이메일·계정 없음. 데이터는 기기(IndexedDB)에만 저장, 서버 전송 없음.
- 광고·분석툴 미적용.
- 위치는 구 수동 선택만(GPS 미사용). 동행자는 유형만(실명 미수집).
- 메모란에 타인 개인정보 입력 금지 안내. 이름·전화·생년월일·성별·주소·직장명 미수집.
- 데이터 통제권: `안내` 화면에서 내보내기(백업)/가져오기(복원)/전체 초기화 직접 수행.

> 개인정보 관련 내용은 일반 정보이며 법률 자문이 아님. 공개 전 개인정보보호위원회(privacy.go.kr) 자료 또는 전문가 확인 권장.

## 데이터 백업 형식

`안내 → 내보내기`로 받는 파일(`오늘의한잔_백업_YYYYMMDD.json`):

```json
{
  "format": "oneul-hanjan-backup",
  "version": 1,
  "exportedAt": "2026-06-02T...Z",
  "profile": { ... },
  "records": [ ... ]
}
```

`가져오기`는 이 파일을 읽어 기존 기기 데이터를 **덮어쓴다**.
